# Directives

This directory contains Standard Operating Procedures (SOPs) written in Markdown.

## Purpose

Directives define:
- **Goals**: What the task should accomplish
- **Inputs**: What data/information is needed
- **Tools/Scripts**: Which execution scripts to use
- **Outputs**: Expected deliverables
- **Edge Cases**: How to handle exceptions

## Guidelines

1. Write directives like instructions for a mid-level employee
2. Be specific about inputs and expected outputs
3. Reference execution scripts by path (e.g., `execution/scrape_single_site.py`)
4. Document edge cases and error handling
5. Update directives when you learn something new (API limits, better approaches, etc.)

## Naming Convention

Use descriptive, lowercase names with underscores:
- `scrape_website.md`
- `generate_report.md`
- `process_data.md`
